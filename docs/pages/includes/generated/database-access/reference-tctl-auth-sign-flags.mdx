| Flag | Description |
| - | - |
| `--user` | Teleport user name |
| `--host` | Teleport host name |
| `-o/--out` | Identity output |
| `--format` | Identity format: file, openssh, tls, kubernetes, db, windows, mongodb, cockroachdb, redis, snowflake, elasticsearch, cassandra, scylla. file is the default. |
| `--ttl` | TTL (time to live) for the generated certificate |
| `--compat` | OpenSSH compatibility flag |
| `--proxy` | Address of the teleport proxy. When --format is set to &#34;kubernetes&#34;, this address will be set as cluster address in the generated kubeconfig file |
| `--overwrite` | Whether to overwrite existing destination files. When not set, user will be prompted before overwriting any existing file. |
| `--leaf-cluster` | Leaf cluster to generate identity file for when --format is set to &#34;kubernetes&#34; |
| `--kube-cluster-name` | Kubernetes cluster to generate identity file for when --format is set to &#34;kubernetes&#34; |
| `--app-name` | Application to generate identity file for. Mutually exclusive with &#34;--db-service&#34;. |
| `--db-service` | Database to generate identity file for. Mutually exclusive with &#34;--app-name&#34;. |
| `--db-user` | Database user placed on the identity file. Only used when &#34;--db-service&#34; is set. |
| `--db-name` | Database name placed on the identity file. Only used when &#34;--db-service&#34; is set. |
| `--windows-user` | Window user placed on the identity file. Only used when --format is set to &#34;windows&#34; |
| `--windows-domain` | Active Directory domain for which this cert is valid. Only used when --format is set to &#34;windows&#34; |
| `--windows-sid` | Optional Security Identifier to embed in the certificate. Only used when --format is set to &#34;windows&#34; |
